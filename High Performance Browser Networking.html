High Performance
Browser Networking.

To optimize the web, you must first understand how a browser retrieves and processes it. This module dissects the Critical Rendering Path (CRP) into 4 chronological stages.

01

The Network Handshake

Before a single byte of HTML is downloaded, the browser must negotiate a connection. Deep dive into DNS resolution algorithms, TCP 3-way handshakes, and TLS encryption costs.

Protocols Latency DNS / TCP / TLS
02

HTTP Data Streaming

Once the secure connection is established, the data flows. Analyze the mechanics of HTTP/2 multiplexing, HTTP/3 over QUIC, and how the browser begins streaming bytes into raw characters.

Data Transfer HTTP/2 & 3 QUIC
03

DOM & CSSOM Construction

The parser takes over. Learn how bytes are converted to characters, tokenized, lexed, and finally constructed into the Document Object Model (DOM) and CSS Object Model (CSSOM) trees.

Parsing Main Thread Tokenization
04

Layout, Paint & Composite

The final visual manifestation. The browser calculates the exact geometry of every node (Layout), rasterizes the pixels (Paint), and sends layers to the GPU (Composite) for high-framerate rendering.

Graphics GPU Acceleration Rasterization
